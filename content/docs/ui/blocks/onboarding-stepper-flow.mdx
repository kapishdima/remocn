A composition scene showing a three-step onboarding flow: account setup, plan selection, and settings configuration. Demonstrates the sanctioned panel-crossfade pattern where step visibility is derived linearly from the stepper's exposed position channel, with no additional motion math.

## Installation

<InstallBlock name="onboarding-stepper-flow" />

<Note>
Installing `onboarding-stepper-flow` automatically installs the shared `remocn-ui` core and all composed primitives via `registryDependencies`: stepper, input, radio, switch, and button. You do not need to install them separately.
</Note>

## Composed primitives

- **Stepper** — multi-step indicator with completed-step checkmarks
- **Input** — text reveal for account name
- **Radio** — plan selection control
- **Switch** — toggle for settings option
- **Button** — next/finish navigation actions

## Timeline narrative

The stepper and step-1 input are both visible at frame 0, with the input beginning to reveal the account name from frame 10 to frame 55. At frame 60, the Next button is pressed, triggering the stepper to advance from position 0 to position 1 from frames 64–90. The panel crossfade is derived linearly from `StepperStyle.position` (opacity = clamp(linear(position))). The radio for plan selection becomes active during its step window (90–104). At frame 100, Next is pressed again, advancing position to 2, and the switch control appears in step 3. At frame 150, the Finish button is pressed and transitions to success, with the stepper simultaneously completing and drawing checkmarks on all passed steps.
