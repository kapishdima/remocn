A composition scene showing the AI request lifecycle: prompt typing, submit, skeleton shimmer, response reveal, and completion toast. The skeleton block's shimmer motion and the crossfade between loading and loaded states demonstrate composition of motion atoms without new motion math.

## Installation

<InstallBlock name="ai-prompt-flow" />

<Note>
Installing `ai-prompt-flow` automatically installs the shared `remocn-ui` core and all composed primitives via `registryDependencies`: input, button, skeleton, skeleton-block, and toast. You do not need to install them separately.
</Note>

## Composed primitives

- **Input** — text reveal for the prompt
- **Button** — hover/press/loading state progression
- **Skeleton** — loading state with shimmer animation
- **Skeleton-block** — motion atom shimmer sweep (loopable)
- **Toast** — completion message auto-dismiss

## Timeline narrative

The input begins revealing the prompt text from frame 0 to frame 50. At frame 52, the cursor moves to the button trigger, entering hover state. Frame 58 brings a press, transitioning to loading at frame 62. Simultaneously, the skeleton enters loading state and skeleton-block begins its shimmer sweep, looping from frame 64 to frame 150. At frame 150, the skeleton state changes to loaded, crossfading from the shimmer to the response text with opacities summing to 1 (no layout shift). The toast appears at frame 160 to confirm the response is ready and auto-dismisses at frame 220.
