# Ralph progress — InteractiveCode (iteration 2: code-editor card redesign)

## Task
Redesign the InteractiveCode section to match the reference mock: a dark
macOS-style code-editor card with traffic-light dots, a component pill (chevron)
top-right, syntax-highlighted <Typewriter/> JSX, INLINE-EDITABLE values
(drag/click numbers, click color swatch, click string/boolean), and a
"Drag or click values to edit" footer hint. (react-bits InteractiveCode pattern.)

## Design decision
Kept the established horizontal layout: interactive code card on the LEFT, live
@remotion/player Typewriter on the RIGHT (stacks on mobile). The code's editable
values ARE the controls and drive the same state feeding the live preview. This
preserves the "tweak it live" value and matches react-bits (code + live output).

## Implemented (single file rewrite)
- app/(home)/components/sections/interactive-code.tsx
  - Dark editor card (#0f0e17, ring, fuchsia corner glow), title bar (3 dots +
    "Typewriter" pill w/ ChevronDown), <pre>/<code> syntax-highlighted JSX,

  - SYNTAX palette (fixed dark theme) via <Token color>; code laid out with

  - Inline editors (all native <button>/<input>, lint-safe, keyboard-accessible):

      (100-900 step 100).
    - EditableString: button -> inline text input for `text`.
    - ColorValue: span wrapper + overlaid native <input type=color> (no handlers
      on span -> no a11y rule), swatch dot + uppercase hex.
    - ToggleBool: button toggling cursor true/false (aria-pressed).
  - set<T>() helper wraps each setState with debounced component_customized analytics.
  - Live preview unchanged: registry["typewriter"] in @remotion/player, inputProps
    from state (cursorColor follows color).

## Key learnings
- biome 2.2.0, recommended + react/next domains; lint = `biome check` (includes
  formatter + organizeImports). Use native <button type="button"> for clickable
  inline elements to avoid useSemanticElements / noStaticElementInteractions; add
  `// biome-ignore lint/a11y/noAutofocus` for click-to-edit autofocus inputs
  (codebase precedent: bento-registry.tsx, tiers.tsx).
- useRef MUST be typed `useRef<X | null>(null)` to stay mutable (the `(null)`
  overload otherwise yields a readonly RefObject).
- Tailwind preflight makes <button> inherit font, so buttons sit inline in the
  mono <pre>; align-baseline keeps them on the text baseline.
- User rule: do NOT run build/dev/verify (incl. biome) — user handles it.
  LSP/tsserver not installed in this env.
