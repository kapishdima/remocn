# remocn

shadcn registry с готовыми анимациями, переходами и backgrounds для Remotion.

## Что это

Набор production-ready компонентов для создания видео в Remotion. Пользователи устанавливают компоненты через `npx shadcn add remocn/<component>` и собирают видео из готовых блоков.

## Целевая аудитория

Solo builders и маленькие команды (1-2 чел), фронтендер знакомый с экосистемой shadcn. Типичный сценарий: сделали продукт → нужно demo video → берут remocn.

## Архитектура

- **Монорепо:** pnpm workspaces + turborepo
- `apps/web/` — Next.js сайт (landing page + Fumadocs для документации)
- `packages/registry/` — shadcn registry manifest + исходники компонентов

## Два уровня компонентов

- **Primitives** — отдельные анимации, переходы, backgrounds
- **Compositions** — готовые сцены, собранные из primitives

## Ключевые решения

- Плоский namespace: `remocn/fade-in`, `remocn/intro-scene`
- Remotion — prerequisite, не bootstrap'им его
- Own your code (shadcn philosophy) — файлы копируются в проект пользователя
- Все компоненты пишутся с нуля на Remotion API (`useCurrentFrame()`, `interpolate()`, `spring()`)
- Вдохновляемся reactbits.dev идеями, но НЕ копируем код (их лицензия MIT + Commons Clause запрещает порт)
- Превью на сайте через `@remotion/player` — интерактивный плеер в браузере
- Лицензия: MIT

## Бизнес-модель

Open core. Free примитивы и базовые compositions (MIT). В будущем — premium блоки и video builder.

## Команды

```bash
pnpm install          # установка зависимостей
pnpm dev              # запуск dev серверов
pnpm build            # сборка всех пакетов
```


## Компоненты

Категория 1: Типографика и Текст (Typography & Text)

Название: <BlurReveal />
Короткое описание: Плавное появление текста с эффектом фокусировки.
Подробное описание эффекта: Текст переходит из полностью прозрачного и сильно размытого состояния в четкое.
Remotion implementation: * Шрифты: Обязательно использование @remotion/google-fonts или @remotion/fonts (локально), чтобы гарантировать загрузку шрифта до рендера кадра, иначе blur отработает на системном шрифте.
Анимация: Используем interpolate() для opacity (0 -> 1) и CSS filter: blur(...). Важно помнить о производительности: сильный blur на больших блоках текста лучше оборачивать в отдельный слой (композицию).

Название: <Typewriter />
Короткое описание: Симуляция печати текста с мигающим курсором.
Подробное описание эффекта: Текст появляется посимвольно. В конце строки мигает курсор.
Remotion implementation:
Детерминированность (Randomness): Для мигания курсора категорически нельзя использовать Math.random() или setInterval. Используем строгую математику: const isCursorVisible = Math.floor(frame / 15) % 2 === 0.

Вывод: Раскрываем массив символов через text.substring(0, Math.floor(interpolate(frame, ...))).
Название: <InlineHighlight />
Короткое описание: Привлечение внимания к конкретному слову в предложении.
Подробное описание эффекта: Выбранное слово плавно меняет цвет.
Remotion implementation: Используем функцию interpolateColors из @remotion/color для плавного перехода от белого (например) к брендовому цвету. Слово оборачивается в <span> внутри текстового блока.

Название: <TextFadeReplace /> / <StrikethroughReplace />
Короткое описание: Замена текста на том же месте.
Подробное описание эффекта: Старый текст исчезает/зачеркивается, новый появляется.
Remotion implementation:
Слои (Layers): Обязательное использование position: absolute в общем контейнере, чтобы тексты не "толкали" друг друга в DOM при смене opacity.
Transitioning: Идеально подходит компонент <Sequence> с отрицательным сдвигом, чтобы анимации затухания старого текста и появления нового пересекались по времени.

Название: <StaggeredFadeUp /> (Слова ступеньками)
Короткое описание: Классическое, мягкое появление текста по одному слову снизу вверх.
Подробное описание эффекта: Каждое слово в предложении плавно всплывает (сдвигается по оси Y вверх) из полной прозрачности. Движение начинается с небольшой задержкой для каждого следующего слова, создавая эффект волны.
Remotion implementation: Текст разбивается на массив слов через .split(' '). Каждое слово оборачивается в <span> с inline-block. Используется interpolate для opacity (0 -> 1) и transform: translateY (например, 20px -> 0px). Задержка высчитывается как frame - index * staggerDelay.

Название: <MaskedSlideReveal /> (Выезд из-за маски)
Короткое описание: Текст буквально «вырастает» из невидимой линии.
Подробное описание эффекта: Буквы или слова появляются, выезжая снизу вверх, но они не просто становятся непрозрачными, а обрезаются по четкой невидимой горизонтальной линии (маске), пока не выедут полностью. Очень строгий и архитектурный эффект.
Remotion implementation: Каждая строка или слово оборачивается в родительский <div> с жестким overflow: hidden и фиксированной высотой (равной line-height). Сам текст внутри анимируется через spring transform: translateY(100%) -> 0%.

Название: <TrackingIn /> (Стягивание интервалов)
Короткое описание: Текст собирается из широкого разброса в нормальное состояние.
Подробное описание эффекта: Слово появляется с огромным расстоянием между буквами (letter-spacing) и сильным размытием, а затем буквы резко «схлопываются» к центру в нормальное слово с легким упругим отскоком.
Remotion implementation: Интерполяция свойства letter-spacing (например, от 0.5em до normal или 0) в комбинации с filter: blur(). Для красивого отскока letter-spacing привязывается к вызову spring().

Название: <ShimmerSweep /> (Блик / Проблеск)
Короткое описание: Световой блик, пробегающий по тексту.
Подробное описание эффекта: Текст написан тусклым цветом (например, темно-серым). В нужный момент по нему слева направо пробегает яркий, светящийся градиент, зажигая буквы и оставляя их яркими. Часто используется для фокуса на AI-фичах (генерация).
Remotion implementation: Использование CSS background-clip: text, color: transparent и background-image: linear-gradient(...). Анимация достигается смещением свойства background-position от -100% до 200% с помощью interpolate по текущему кадру.

Название: <MarkerHighlight /> (Маркерное выделение)
Короткое описание: Имитация выделения текста желтым маркером или цветным блоком.
Подробное описание эффекта: За целевым словом или фразой плавно, слева направо, отрисовывается цветной прямоугольник. Текст при этом может менять цвет на контрастный (например, с черного на белый).
Remotion implementation: Псевдоэлемент ::after или абсолютно спозиционированный <div> позади текста с z-index: -1. Анимируется его width от 0% до 100% через spring с небольшим damping для реалистичности.

Название: <SlotMachineRoll /> (Вращение барабана)
Короткое описание: Буквы или цифры крутятся по вертикали, как в казино.
Подробное описание эффекта: При смене данных (например, цены с $99 на $199 или статуса с "Pending" на "Success") символы быстро прокручиваются сверху вниз, пока не зафиксируются на новом значении.
Remotion implementation: Контейнер с overflow: hidden высотой в одну строку. Внутри — длинная колонка (или <div>) со всеми возможными символами/цифрами (или просто начальным и конечным). Колонка анимируется по оси Y (translateY) так, чтобы нужный символ остановился точно по центру.

Название: <MatrixDecode /> (Расшифровка / Scramble)
Короткое описание: Беспорядочные символы превращаются в осмысленное слово.
Подробное описание эффекта: Слово появляется как набор случайных букв, цифр и спецсимволов (#@%*!). Затем, символ за символом (слева направо), этот «мусор» фиксируется и превращается в правильный текст. Идеально для DevTools и кибербезопасности.
Remotion implementation: Хук, который на каждом кадре генерирует строку нужной длины. Для символов, которые еще "не расшифрованы", берется случайный символ из заранее заданного словаря (используя псевдослучайный генератор @remotion/random на основе frame, чтобы избежать мерцания при рендере). Для "расшифрованных" — берется оригинальный символ.

Название: <RGBGlitchText />
Короткое описание: Текст дергается и расслаивается на красные, зеленые и синие каналы.
Подробное описание эффекта: В момент резкого перехода или звукового акцента текст на долю секунды смещается, оставляя за собой RGB-шлейфы (хроматическая аберрация) и искажаясь по горизонтали.
Remotion implementation: Создается 3 копии одного и того же текста, наложенных друг на друга (position: absolute). У одной копии цвет текста (или text-shadow) красный, у другой синий, у третьей зеленый. С помощью interpolate и random копии смещаются на 2-5 пикселей в разные стороны на очень короткий промежуток времени (3-5 кадров), а затем возвращаются в центр.

Название: <VideoMaskText /> (Текст как окно)
Короткое описание: Гигантский текст, внутри которого проигрывается видео продукта.
Подробное описание эффекта: На экране огромное жирное слово (например, "CREATE"). Буквы прозрачные, а фон вокруг них черный. Сквозь буквы видно, как работает интерфейс. Затем камера резко "влетает" в букву, и видео интерфейса открывается на весь экран.
Remotion implementation: Использование CSS-свойства mask-image (или <clipPath> в SVG). Текст выступает маской для видео-компонента или скринкаста. Пролет камеры реализуется через колоссальное увеличение масштаба (transform: scale(100)) с точным transform-origin по центру нужной буквы.

Название: <InfiniteMarquee /> (Бегущая строка)
Короткое описание: Непрерывно ползущая строка из слов или интеграций.
Подробное описание эффекта: Лента текста, которая бесконечно и плавно движется по горизонтали. Часто используется крупным, контурным шрифтом (stroke) на фоне.
Remotion implementation: Контейнер с текстом дублируется 2-3 раза в ряд. Анимируется transform: translateX всего родителя от 0% до -100% (ширина одного оригинального блока) линейно через interpolate. При достижении -100% анимация зацикливается, визуально создавая бесконечный поток.

Категория 2: Фоны и Атмосфера (Backgrounds)

Название: <MeshGradientBg />
Короткое описание: Живой, плавно переливающийся градиентный фон.
Подробное описание эффекта: Аморфные цветовые пятна медленно перемещаются, меняя форму.
Remotion implementation (Продвинутая): * Вместо тяжелых CSS-блюров идеальным решением будет @remotion/noise (Noise Visualization). Мы можем использовать noise2D или noise3D (где третьим измерением будет frame / скорость), чтобы генерировать органичные, детерминированные координаты смещения для SVG-пятен.
Альтернатива для high-end: использование Shaders (<WebGL>). Remotion поддерживает написание кастомных GLSL шейдеров для фона. Это даст 60FPS при любой сложности градиента. Можно также подмешать готовые эффекты из раздела Light Leaks для создания кинематографичных засветок.

Название: <DynamicGrid />
Короткое описание: Тонкая фоновая сетка (или точки).
Подробное описание эффекта: Медленно движущаяся сетка.
Remotion implementation: CSS linear-gradient или SVG <pattern>. Для движения интерполируем transform: translate() контейнера с сеткой (согласно transforms.md, анимировать translate дешевле для браузера, чем background-position).

Категория 3: Базовые UI-примитивы (UI Elements)

Название: <SpringPopIn />
Короткое описание: Эластичное появление элемента.
Подробное описание эффекта: Увеличение масштаба с легким "отскоком" (overshoot).
Remotion implementation:
Используем spring({ fps, frame, config: { damping: 12 } }).
Transforms: Применяем как transform: scale(${scaleValue}). Важно задать transform-origin: center (или нужный угол).

Название: <SimulatedCursor />
Короткое описание: Движение мыши и клики.
Подробное описание эффекта: Курсор перемещается между точками.
Remotion implementation:
Ассеты: Иконка курсора должна лежать в папке public/ и загружаться через staticFile('cursor.svg') (assets.md), чтобы избежать проблем с бандлингом.
Слои: Курсор должен находиться в самом корневом компоненте сцены с максимальным z-index (или рендериться последним в DOM) и position: absolute.

Название: <PulsingIndicator />
Короткое описание: Состояние загрузки.
Подробное описание эффекта: Непрерывная пульсация.
Remotion implementation: Использование Math.sin(frame / скорость) для генерации цикличного значения от -1 до 1, которое затем мапится через интерполяцию в scale от 0.9 до 1.1.

Категория 4: Сложные композиции (Layouts & Compositions)

Название: <StaggeredBentoGrid />
Короткое описание: Каскадное появление сетки карточек.
Подробное описание эффекта: Карточки появляются по очереди.
Remotion implementation:
Transitioning: Идеальный юзкейс для <Series> или маппинга дочерних компонентов в <Sequence from={index * 15}>. Это гарантирует, что каждый компонент начнет свой жизненный цикл (useCurrentFrame внутри него начнется с 0) в нужное время, делая код чистым.

Название: <ChatToPreviewLayout />
Короткое описание: Разделение экрана.
Подробное описание эффекта: Чат сужается, предпросмотр расширяется.
Remotion implementation: Flexbox с интерполяцией flex-basis или width в процентах. Важно задать жесткие условия overflow: hidden для дочерних контейнеров, чтобы контент не ломал верстку во время изменения ширины родителя.

Название: <BoundingBoxSelector />
Короткое описание: Симуляция интерфейса редактирования (Figma-style).
Подробное описание эффекта: Появление рамки выделения.
Remotion implementation: Для точного позиционирования рамки вокруг переданного children можно использовать хук measuring.md, если размер неизвестен заранее, или требовать от пользователя передачи явных width / height пропсов для производительности.

Категория 5: Переходы и Камера (Transitions & Camera)

Название: <DirectionalWipe /> (и Slide/Pan)
Короткое описание: Классический сдвиг экрана.
Подробное описание эффекта: Смена сцен сдвигом.
Remotion implementation:
Transitioning: Две сцены оборачиваются в пересекающиеся <Sequence>.
Transforms: Согласно документации, используем только transform: translateX/Y, а не left/top, чтобы не триггерить пересчет Layout (Reflow) в браузере при рендере кадров.

Название: <DeviceMockupZoom />
Короткое описание: Отдаление от UI к устройству.
Подробное описание эффекта: Интерфейс отдаляется и оказывается экраном ноутбука/телефона.
Remotion implementation:
Рамки мокапов (png/webp с прозрачным экраном) загружаем через staticFile() (assets.md).
Интерфейс пользователя помещается "под" рамку (z-index или порядок в DOM). Затем весь родительский контейнер с мокапом анимируется через transform: scale().

Название: <ZoomThroughTransition />
Короткое описание: Пролет "сквозь" элемент.
Подробное описание эффекта: Резкое увеличение сцены.
Remotion implementation: Интерполяция scale с сильным ускорением (экспоненциальным или с помощью агрессивного easing). Крайне важно использовать transform-origin, нацеленный точно на центр элемента, "сквозь" который мы пролетаем.

Категория 6: Визуализация данных (Data Visualization)

Название: <AnimatedLineChart /> / <AnimatedBarChart />
Короткое описание: Оживающие графики для дашбордов.
Подробное описание эффекта: Линия графика плавно отрисовывается слева направо. Столбцы гистограммы упруго "вырастают" снизу вверх. Цифры на осях могут "накручиваться" (Count-up).
Remotion implementation: * Линии: Использование SVG <path>. Анимация достигается через интерполяцию свойства stroke-dashoffset от длины пути (можно посчитать заранее или через getTotalLength() в useEffect) до 0.
Столбцы: spring анимация scaleY для <rect>, с обязательным transform-origin: bottom.
Математика: Использование remotion/animation-math для плавного накручивания чисел (count-up).

Категория 7: Специфичные Dev-блоки (Dev & CLI)

Название: <TerminalSimulator />
Короткое описание: Окно консоли с выводом логов.
Подробное описание эффекта: Отличается от <Typewriter>. Здесь симулируется выполнение команд (например, npm run build): текст появляется строками, а когда окно заполняется, старые строки "уезжают" вверх, освобождая место для новых.
Remotion implementation: * Массив объектов { text: string, type: 'command' | 'log' | 'success', delay: number }.
Отрисовка новых строк через <Sequence>.
Симуляция скролла: родительский контейнер сдвигается вверх (transform: translateY) на высоту строки каждый раз, когда добавляется новая строка за пределами видимости.

Название: <CodeDiffWipe /> (Before/After)
Короткое описание: Сравнение "До" и "После".
Подробное описание эффекта: На экране старый, громоздкий код (или UI). Поверх него плавно скользит вертикальная или диагональная линия (часто с ползунком), открывая новый, чистый код (или результат работы продукта).
Remotion implementation: * Два слоя (компонента), наложенных друг на друга (position: absolute).
Верхний слой анимируется через CSS clip-path: inset(0 X% 0 0) (или polygon), где X интерполируется от 100 до 0 с помощью spring или плавной interpolate.

Категория 8: Микроинтеракции (Micro-interactions)

Название: <ToastNotification />
Короткое описание: Всплывающее системное уведомление.
Подробное описание эффекта: Небольшая плашка (например, "Deployment successful") выскакивает в углу экрана, висит несколько секунд и исчезает.
Remotion implementation:
Компонент с абсолютным позиционированием (например, bottom-4 right-4).
Появление через spring (translateY вверх + opacity).
Исчезновение через interpolate (translateY вниз + opacity), используя useCurrentFrame() относительно конца длительности компонента (вычисляется из переданного пропса durationInFrames).

Название: <DragAndDropFlow />
Короткое описание: Симуляция загрузки файла.
Подробное описание эффекта: Файл (иконка документа/картинки) "перетаскивается" курсором в зону загрузки, зона подсвечивается (меняет границу/цвет), файл отпускается, и начинается анимация загрузки (прогресс-бар).
Remotion implementation:
Объединение <SimulatedCursor> и компонента зоны загрузки.
Синхронизация по кадрам: когда координаты курсора совпадают с зоной загрузки, зона использует interpolateColors для подсветки. После "клика" (отпускания) запускается Sequence с анимацией <PulsingIndicator> или растущего width у прогресс-бара.

Категория 9: Процессы и Статусы (Workflows)

Название: <ProgressSteps /> / <Timeline />
Короткое описание: Визуализация многошагового процесса.
Подробное описание эффекта: Горизонтальная или вертикальная линия с точками (шагами). Точки последовательно "зажигаются" (меняют цвет), а линия между ними "заполняется" цветом, показывая движение данных или процесс сборки (например, "Connect -> Process -> Deploy").
Remotion implementation:
Использование <Sequence> для координации старта каждого шага.
Линии между шагами — это SVG <line>, которые закрашиваются через анимацию stroke-dashoffset (как в графиках).
Сами точки пульсируют (spring scale) при переходе в активное состояние.

Название: <SuccessConfetti />
Короткое описание: Радостный финал успешного действия.
Подробное описание эффекта: Взрыв конфетти или партиклов из определенной точки (например, после нажатия кнопки "Merge" или "Publish").
Remotion implementation:
В документации Remotion есть отличные примеры работы с canvas для частиц.
Обязательное условие: использование random(seed) из @remotion/random для генерации векторов полета частиц. Обычный Math.random() сломает консистентность при рендере видео. Партиклы рендерятся в <canvas> внутри компонента Remotion, где координаты пересчитываются на каждый кадр.



Название: <CursorFlow />
Короткое описание: Реалистичная симуляция движения и кликов мыши.
Подробное описание эффекта: Курсор не просто скользит по прямым линиям. Он имитирует биомеханику человеческой руки: плавно ускоряется в начале, двигается по легкой дуге, замедляется перед целью (easing), делает микро-паузу "прицеливания", а при самом клике визуально "вдавливается" в экран.
Remotion implementation (Продвинутая):
Вместо линейной interpolate между двумя точками (x1, y1) -> (x2, y2), необходимо использовать кубические кривые Безье (Bezier curves) для расчета координат x и y на каждом кадре. Можно использовать функции из d3-shape для генерации сплайнов по массиву waypoints (ключевых точек).
Сам курсор — это SVG-компонент, позиционированный абсолютно (z-index: 9999).
Анимация клика реализуется через независимый spring() на свойство scale, который триггерится в кадры, указанные в конфиге компонента.
Taste detail: Никаких прямых углов и роботизированного движения! Траектория всегда должна иметь "животик" (дугу). При клике курсор должен резко уменьшаться (scale: 0.85) буквально на 2-3 кадра с жестким damping, создавая ощущение физического удара по стеклу, а элемент, по которому кликнули, должен синхронно "прожаться" на scale: 0.98.

Название: <GlassCodeBlock /> (или CodeWindow)
Короткое описание: Премиальное окно редактора кода с эффектом матового стекла.
Подробное описание эффекта: Окно терминала или редактора, которое не выглядит как глухой черный квадрат. Оно имеет объем, отражает свет по краям (Linear-style glow) и слегка просвечивает анимированный фон (MeshGradientBg), находящийся за ним. Текст кода внутри появляется с идеальным подсветкой синтаксиса.
Remotion implementation (Продвинутая):
Фон: Контейнер использует background: rgba(10, 10, 10, 0.6) в связке с backdrop-filter: blur(16px) и WebkitBackdropFilter.
Микробордер: Не используйте обычный CSS border. Вместо этого оберните окно в родительский div с padding: 1px (или используйте ::before псевдоэлемент) и задайте ему background: linear-gradient(180deg, rgba(255,255,255,0.15) 0%, rgba(255,255,255,0.0) 100%). Это создаст эффект света, падающего точно на верхнюю грань окна.
Taste detail: Три точки управления окном (macOS traffic lights) в левом верхнем углу не должны "кричать" яркими цветами. Сделайте их слегка приглушенными (opacity: 0.6) и добавьте внутреннюю микро-тень всему окну (box-shadow: inset 0 1px 0 rgba(255,255,255,0.05)), чтобы оно казалось слегка вдавленным в холст, как дорогой кусок пластика.

Название: <DynamicSplitScreen /> (ChatToPreviewLayout)
Короткое описание: Кинематографичное разделение экрана на две рабочие зоны.
Подробное описание эффекта: Сцена начинается с одного широкого блока по центру (например, чат-интерфейс). По мере развития сюжета, этот блок плавно уезжает влево, сжимаясь, а справа, как бы вытягиваясь из-под маски, появляется новая рабочая панель (результат генерации или предпросмотр).
Remotion implementation (Продвинутая):
Родитель — Flexbox-контейнер (display: flex).
Левый блок: интерполяция свойства width (или flex-basis) от 100% до 35%.
Правый блок: интерполяция от 0% до 65%.
Критично: Чтобы верстка внутри колонок не ломалась во время сжатия (текст не прыгал по строкам), оба дочерних контейнера должны иметь фиксированную внутреннюю ширину (например, min-width: 800px) и строгое правило overflow: hidden.
Taste detail: Используйте нелинейное, драматичное замедление (например, Easing.bezier(0.16, 1, 0.3, 1) — Apple's custom ease). Во время сдвига правый (новый) блок должен слегка "выцветать" (opacity: 0 -> 1) и сдвигаться по оси X навстречу своему росту (translateX: 40px -> 0px). Это создаст иллюзию, что левый блок физически "тащит" правый за собой.

Название: <TerminalSimulator />
Короткое описание: Симуляция сборки проекта в консоли (CLI).
Подробное описание эффекта: Строки логов (Downloading, Building, Resolving) появляются не плавно, а отрывисто, блоками — имитируя реальную задержку сети или процессора. Когда строк становится больше, чем высота окна, старые строки "уезжают" вверх жестким, ступенчатым рывком.
Remotion implementation (Продвинутая):
Компонент принимает массив объектов { text, type (success/warn/error), durationInFrames }.
Используется Math.floor() в комбинации с interpolate, чтобы выводить текст не посимвольно (как в <Typewriter>), а целыми кусками или строками.
Скролл: рассчитывается высота одной строки (line-height). Если количество отрендеренных строк превышает лимит, родительский контейнер сдвигается через transform: translateY на величину -N * lineHeight.
Taste detail: Аутентичность — в несовершенстве. Включите микро-задержки (freeze frames) на строках типа Resolving dependencies..., чтобы создать напряжение перед быстрым выплеском строк лога об успешной сборке. Не используйте spring для скролла консоли! Терминал скроллится жестко, мгновенными прыжками (step function).

Название: <MorphingModal />
Короткое описание: Трансформация кнопки или карточки в полноэкранное окно.
Подробное описание эффекта: Пользователь кликает на маленькую карточку в сетке (Bento Grid). Карточка не исчезает; она "отрывается" от сетки, взмывает над остальным интерфейсом (появляется мощная тень) и плавно растягивается, пока не заполнит весь экран, обнажая детальный контент внутри себя.
Remotion implementation (Продвинутая):
Требуется абсолютное позиционирование. Компонент должен "знать" свои начальные координаты (Top/Left/Width/Height карточки) и конечные координаты (0/0/100%/100% экрана).
Используется сложный, тяжеловесный spring для одновременной интерполяции 5 свойств: top, left, width, height и border-radius (например, от 24px до 0px).
Фон за модалкой должен синхронно затемняться (background: rgba(0,0,0,0) -> rgba(0,0,0,0.8)).
Taste detail: Это самая "дорогая" анимация в арсенале. Чтобы она не казалась дешевым слайд-шоу, контент внутри карточки-источника должен плавно исчезать (opacity -> 0) в первой трети анимации, а контент полноэкранного окна должен появляться (opacity -> 1, translateY: 20px -> 0) только в последней трети, когда окно почти достигло своих финальных размеров. Ощущение массы: spring должен иметь высокий mass и низкий stiffness, чтобы окно "раскрывалось" величественно, а не дергалось.


Название: <SpotlightCard /> (Эффект фонарика / Magnetic Glow)
Короткое описание: Имитация света, который следует за курсором, подсвечивая края и фон карточки.
Подробное описание эффекта: Карточка (bento-блок) лежит в темноте. Когда виртуальный курсор <SimulatedCursor> пролетает над ней или рядом, за ним тянется мягкий, радиальный источник света, который "выхватывает" из темноты тончайший микробордер карточки и слегка подсвечивает ее внутренности.
Remotion implementation (Продвинутая):
Карточка должна "слушать" координаты виртуального курсора (передаются через React Context или пропсы).
Свечение на фоне: background: radial-gradient(800px circle at ${mouseX}px ${mouseY}px, rgba(255,255,255,0.06), transparent 40%).
Свечение на рамке: Псевдоэлемент ::before с padding: 1px (как рамка) и аналогичным radial-gradient, но более ярким rgba(255,255,255,0.3). Накладывается через mask-image, чтобы светилась только граница.
Taste detail: Свет никогда не должен быть цветным, если только это не бренд-цвет с opacity: 0.05. Это должен быть "лунный", холодный свет. Свечение должно иметь небольшую инерцию (пружинить за курсором с микро-задержкой в 1 кадр), чтобы казаться органичным.

Название: <VolumetricRays /> (Объемные лучи / God Rays)
Короткое описание: Кинематографичные лучи света, пробивающиеся сквозь текст или логотип.
Подробное описание эффекта: Эффект "маяка". За крупным заголовком или логотипом находится источник света. Свет пробивается сквозь контуры букв, создавая длинные, медленно вращающиеся или пульсирующие лучи в пыльной/туманной среде.
Remotion implementation (Продвинутая):
В вебе это крайне тяжело сделать чисто на CSS без потери FPS. В Remotion это реализуется через многослойность.
Дублируем текст. Задний слой размываем (огромный filter: blur()) и применяем transform: scaleY(1.5) scaleX(1.2) с transform-origin: center bottom.
Добавляем <WebGL> шейдер поверх размытого текста, который накладывает радиальный шум (@remotion/noise), маскируя "лучи" так, чтобы они казались неоднородными, словно проходят сквозь дым.
Taste detail: Лучи не должны быть статичными. Используй очень медленный interpolate для вращения (rotate) или смещения центра источника света. Текст спереди должен оставаться абсолютно черным или темным (силуэт), чтобы контраст создавал глубину.

Название: <DataFlowPipes /> (Импульсы данных / Cyber Routing)
Короткое описание: Визуализация передачи пакетов данных по трубкам/проводам системы.
Подробное описание эффекта: Часто используется для показа API, вебхуков или баз данных. На экране сложная схема из тонких линий. По этим линиям стремительно проносятся светящиеся "импульсы" (капли света), оставляя за собой затухающий хвост.
Remotion implementation (Продвинутая):
Строим сеть через SVG <path>.
Импульс создается путем клонирования <path> поверх оригинального, назначения ему яркого цвета и добавления stroke-dasharray="50 1000" (где 50 — длина импульса, 1000 — пустота).
Анимируем stroke-dashoffset от длины пути до 0, чтобы импульс пробежал от начала до конца.
Для хвоста импульса: используем SVG <filter> с <feGaussianBlur> и <feColorMatrix>, либо создаем несколько импульсов с падающей opacity, летящих друг за другом с задержкой в 1-2 кадра.
Taste detail: Скорость импульсов должна быть неравномерной. Они должны быстро "выстреливать" (ease-in), лететь по прямым участкам и слегка притормаживать на изгибах 90 градусов (используя кастомный cubic-bezier).

Название: <CodeAccordion /> (Интеллектуальное сворачивание)
Короткое описание: Длинный блок кода сжимается, скрывая нерелевантные строки.
Подробное описание эффекта: У нас есть блок кода на 50 строк. Камера фокусируется, и строки с 10 по 45 внезапно "схлопываются" по вертикали, заменяясь на элегантную плашку // ... 35 lines collapsed ..., оставляя видимыми только начало функции и ее конец.
Remotion implementation (Продвинутая):
Код должен быть отрендерен не как единый текст, а как массив строк (<div> на каждую строку).
Строки, которые нужно скрыть, оборачиваются в контейнер с overflow: hidden.
Используем spring() для интерполяции height этого контейнера от 100% до 0px, синхронно уводя opacity содержимого в 0. Плашка // ... появляется через встречный fade-in.
Taste detail: Когда код "схлопывается", он должен иметь легкий физический вес. Родительский контейнер (само окно редактора) должно синхронно уменьшить свою высоту с упругим spring, чтобы нижняя рамка "ударилась" о верхнюю часть кода с микро-отскоком (damping: 10).

Название: <PerspectiveMarquee /> (3D Бегущая строка)
Короткое описание: Лента логотипов, уходящая вдаль в перспективу.
Подробное описание эффекта: Бегущая строка (например, "Trusted by..."), которая крутится не плоско по экрану, а повернута под углом в 3D пространстве. Логотипы, которые находятся ближе к зрителю — резкие и крупные, а те, что уезжают вдаль — размываются и растворяются во тьме.
Remotion implementation (Продвинутая):
Контейнер-бесконечная лента анимируется через transform: translateX().
Родительский контейнер имеет transform: rotateY(-30deg) rotateX(10deg) и perspective: 1000px.
Секрет глубины (Depth of Field): накладываем абсолютно позиционированный <div /> поверх всего этого блока. Его фон: linear-gradient(90deg, transparent 0%, transparent 60%, rgba(0,0,0,1) 100%).
В идеале: логотипы должны динамически менять filter: blur() в зависимости от их X координаты на экране (чем дальше X от центра перспективы, тем больше blur).
Taste detail: Скорость ленты должна быть гипнотически медленной. Размытие логотипов на заднем плане не должно быть "дешевым". Используйте многоступенчатый градиент в маске, чтобы они растворялись в черном фоне абсолютно бесшовно, словно в тумане.



Название: <FrostedGlassWipe /> (Стеклянная шторка)
Короткое описание: Элегантный переход через матовое стекло.
Подробное описание эффекта: Вместо того чтобы одна сцена просто перекрывала другую, между ними проезжает панель "матового стекла". Старая сцена размывается под стеклом, а когда стекло уезжает, под ним уже находится новая, четкая сцена.
Remotion implementation (Продвинутая):

Taste detail: Стекло должно иметь физические свойства. Добавьте панели левую и правую микро-границы (border-left: 1px solid rgba(255,255,255,0.2)), чтобы она ловила "свет", когда скользит поверх темного интерфейса. Скорость скольжения должна быть неравномерной (быстрый старт, небольшое замедление в центре, быстрый вылет).

Название: <KineticTypeMask /> (Типографическая маска)
Короткое описание: Использование огромного текста в качестве "окна" в следующую сцену.
Подробное описание эффекта: На экране появляется гигантский заголовок (например, "NEXT" или "02"). Текст абсолютно черный. За текстом — следующая сцена. Затем текст стремительно увеличивается до тех пор, пока внутреннее пространство одной из букв (например, кольцо буквы "O") не расширится за пределы экрана, полностью открыв новую сцену.
Remotion implementation (Продвинутая):
Использование CSS mask-image или SVG <clipPath> с текстом.
Новая сцена находится поверх старой, но скрыта маской текста.
Интерполяция scale текста начинается с нормального размера и экспоненциально (через агрессивный easing) уходит в огромные значения (например, scale: 100).

Taste detail: Используйте только самые жирные начертания шрифта (Geist Black / 900). Перед началом масштабирования дайте зрителю прочитать текст (пауза 10-15 кадров).

Название: <SpatialPush /> (Пространственное выдавливание)
Короткое описание: Новая сцена физически "придавливает" старую.
Подробное описание эффекта: Старая сцена не просто исчезает, она реагирует на появление новой. Старый экран сжимается (scale) и уходит на задний план, а новый экран тяжело падает сверху или съезжает сбоку, накрывая его огромной тенью.
Remotion implementation (Продвинутая):
Сцена А (Старая): В момент начала перехода интерполируем transform: scale(1) -> scale(0.92) и filter: brightness(1) -> brightness(0.5). border-radius также должен плавно увеличиться с 0 до 16px.
Сцена Б (Новая): Въезжает через spring() с translateY(100%) -> translateY(0%).

Taste detail: Эффект веса. Сцена Б не должна останавливаться как вкопанная. Используйте spring с небольшим overshoot (проскоком) — пусть новая сцена пролетит чуть выше нуля (-5px) и с глухим ударом вернется на 0. Сцена А в этот момент должна микро-дернуться, передавая кинетическую энергию удара.

Название: <GridPixelateWipe /> (Пикселизация / Сборка)
Короткое описание: Сцена рассыпается на квадраты (bento-пиксели) и собирается в новую.
Подробное описание эффекта: Вместо плавного fade, экран разбивается на сетку блоков. Блоки старой сцены быстро гаснут (или поворачиваются в 3D) в случайном или диагональном порядке, открывая под собой блоки новой сцены.
Remotion implementation (Продвинутая):
Это ресурсоемкий эффект. Вместо реального разрезания DOM на куски, используем CSS маску (Grid-based mask).
Создаем поверх экрана <div />, состоящий из CSS-сетки (например, 10x10). Каждая ячейка сетки залита черным цветом (выступает маской).
С помощью хука useCurrentFrame и @remotion/random анимируем opacity или transform: scale(0) каждой отдельной ячейки с небольшим каскадным сдвигом (stagger).
Taste detail: Для стиля Vercel сетка должна быть идеальной. Не используйте чистый random для времени затухания квадратов; используйте детерминированную математику (например, задержка = (x * y) % 15), чтобы квадраты исчезали по красивым геометрическим паттернам (волна, диагональ, спираль).

Название: <ChromaticAberrationWipe /> (Глитч-сдвиг)
Короткое описание: Экстремально быстрый переход с RGB-расслоением.
Подробное описание эффекта: Идеально для быстрых демо-роликов DevTools (вспомни видео Vercel). Сцена срывается влево, но на пике скорости она "не выдерживает" и расслаивается на красный, синий и зеленый каналы по краям экрана.
Remotion implementation (Продвинутая):
Обычный переход transform: translateX(), но выполняющийся очень быстро (5-8 кадров).
В момент пиковой скорости (например, кадры 3, 4 и 5) добавляем к контейнеру CSS-свойство filter: drop-shadow(-8px 0 0 rgba(255,0,0,0.8)) drop-shadow(8px 0 0 rgba(0,255,255,0.8)).
Как только скорость падает, drop-shadow должен мгновенно исчезнуть.
Taste detail: Главное правило глитча — его не должно быть много. Эффект хроматической аберрации должен длиться ровно столько, чтобы зритель почувствовал скорость подсознательно, но не успел разглядеть цвета. 3-4 кадра максимум.
