# remocn

shadcn registry с готовыми анимациями, переходами и backgrounds для Remotion.

## Что это

Набор production-ready компонентов для создания видео в Remotion. Пользователи устанавливают компоненты через `npx shadcn add remocn/<component>` и собирают видео из готовых блоков.

## Целевая аудитория

Solo builders и маленькие команды (1-2 чел), фронтендер знакомый с экосистемой shadcn. Типичный сценарий: сделали продукт → нужно demo video → берут remocn.

## Архитектура

- **Монорепо:** pnpm workspaces + turborepo
- `apps/web/` — Next.js сайт (landing page + Fumadocs для документации)
- `packages/registry/` — shadcn registry manifest + исходники компонентов

## Два уровня компонентов

- **Primitives** — отдельные анимации, переходы, backgrounds
- **Compositions** — готовые сцены, собранные из primitives

## Ключевые решения

- Плоский namespace: `remocn/fade-in`, `remocn/intro-scene`
- Remotion — prerequisite, не bootstrap'им его
- Own your code (shadcn philosophy) — файлы копируются в проект пользователя
- Все компоненты пишутся с нуля на Remotion API (`useCurrentFrame()`, `interpolate()`, `spring()`)
- Вдохновляемся reactbits.dev идеями, но НЕ копируем код (их лицензия MIT + Commons Clause запрещает порт)
- Превью на сайте через `@remotion/player` — интерактивный плеер в браузере
- Лицензия: MIT

## Бизнес-модель

Open core. Free примитивы и базовые compositions (MIT). В будущем — premium блоки и video builder.

## Команды

```bash
pnpm install          # установка зависимостей
pnpm dev              # запуск dev серверов
pnpm build            # сборка всех пакетов
```


## Компоненты

Категория 1: Типографика и Текст (Typography & Text)

Название: <BlurReveal />
Короткое описание: Плавное появление текста с эффектом фокусировки.
Подробное описание эффекта: Текст переходит из полностью прозрачного и сильно размытого состояния в четкое.
Remotion implementation: * Шрифты: Обязательно использование @remotion/google-fonts или @remotion/fonts (локально), чтобы гарантировать загрузку шрифта до рендера кадра, иначе blur отработает на системном шрифте.
Анимация: Используем interpolate() для opacity (0 -> 1) и CSS filter: blur(...). Важно помнить о производительности: сильный blur на больших блоках текста лучше оборачивать в отдельный слой (композицию).

Название: <Typewriter />
Короткое описание: Симуляция печати текста с мигающим курсором.
Подробное описание эффекта: Текст появляется посимвольно. В конце строки мигает курсор.
Remotion implementation:
Детерминированность (Randomness): Для мигания курсора категорически нельзя использовать Math.random() или setInterval. Используем строгую математику: const isCursorVisible = Math.floor(frame / 15) % 2 === 0.

Вывод: Раскрываем массив символов через text.substring(0, Math.floor(interpolate(frame, ...))).
Название: <InlineHighlight />
Короткое описание: Привлечение внимания к конкретному слову в предложении.
Подробное описание эффекта: Выбранное слово плавно меняет цвет.
Remotion implementation: Используем функцию interpolateColors из @remotion/color для плавного перехода от белого (например) к брендовому цвету. Слово оборачивается в <span> внутри текстового блока.

Название: <TextFadeReplace /> / <StrikethroughReplace />
Короткое описание: Замена текста на том же месте.
Подробное описание эффекта: Старый текст исчезает/зачеркивается, новый появляется.
Remotion implementation:
Слои (Layers): Обязательное использование position: absolute в общем контейнере, чтобы тексты не "толкали" друг друга в DOM при смене opacity.
Transitioning: Идеально подходит компонент <Sequence> с отрицательным сдвигом, чтобы анимации затухания старого текста и появления нового пересекались по времени.

Категория 2: Фоны и Атмосфера (Backgrounds)

Название: <MeshGradientBg />
Короткое описание: Живой, плавно переливающийся градиентный фон.
Подробное описание эффекта: Аморфные цветовые пятна медленно перемещаются, меняя форму.
Remotion implementation (Продвинутая): * Вместо тяжелых CSS-блюров идеальным решением будет @remotion/noise (Noise Visualization). Мы можем использовать noise2D или noise3D (где третьим измерением будет frame / скорость), чтобы генерировать органичные, детерминированные координаты смещения для SVG-пятен.
Альтернатива для high-end: использование Shaders (<WebGL>). Remotion поддерживает написание кастомных GLSL шейдеров для фона. Это даст 60FPS при любой сложности градиента. Можно также подмешать готовые эффекты из раздела Light Leaks для создания кинематографичных засветок.

Название: <DynamicGrid />
Короткое описание: Тонкая фоновая сетка (или точки).
Подробное описание эффекта: Медленно движущаяся сетка.
Remotion implementation: CSS linear-gradient или SVG <pattern>. Для движения интерполируем transform: translate() контейнера с сеткой (согласно transforms.md, анимировать translate дешевле для браузера, чем background-position).

Категория 3: Базовые UI-примитивы (UI Elements)

Название: <SpringPopIn />
Короткое описание: Эластичное появление элемента.
Подробное описание эффекта: Увеличение масштаба с легким "отскоком" (overshoot).
Remotion implementation:
Используем spring({ fps, frame, config: { damping: 12 } }).
Transforms: Применяем как transform: scale(${scaleValue}). Важно задать transform-origin: center (или нужный угол).

Название: <SimulatedCursor />
Короткое описание: Движение мыши и клики.
Подробное описание эффекта: Курсор перемещается между точками.
Remotion implementation:
Ассеты: Иконка курсора должна лежать в папке public/ и загружаться через staticFile('cursor.svg') (assets.md), чтобы избежать проблем с бандлингом.
Слои: Курсор должен находиться в самом корневом компоненте сцены с максимальным z-index (или рендериться последним в DOM) и position: absolute.

Название: <PulsingIndicator />
Короткое описание: Состояние загрузки.
Подробное описание эффекта: Непрерывная пульсация.
Remotion implementation: Использование Math.sin(frame / скорость) для генерации цикличного значения от -1 до 1, которое затем мапится через интерполяцию в scale от 0.9 до 1.1.

Категория 4: Сложные композиции (Layouts & Compositions)

Название: <StaggeredBentoGrid />
Короткое описание: Каскадное появление сетки карточек.
Подробное описание эффекта: Карточки появляются по очереди.
Remotion implementation:
Transitioning: Идеальный юзкейс для <Series> или маппинга дочерних компонентов в <Sequence from={index * 15}>. Это гарантирует, что каждый компонент начнет свой жизненный цикл (useCurrentFrame внутри него начнется с 0) в нужное время, делая код чистым.

Название: <ChatToPreviewLayout />
Короткое описание: Разделение экрана.
Подробное описание эффекта: Чат сужается, предпросмотр расширяется.
Remotion implementation: Flexbox с интерполяцией flex-basis или width в процентах. Важно задать жесткие условия overflow: hidden для дочерних контейнеров, чтобы контент не ломал верстку во время изменения ширины родителя.

Название: <BoundingBoxSelector />
Короткое описание: Симуляция интерфейса редактирования (Figma-style).
Подробное описание эффекта: Появление рамки выделения.
Remotion implementation: Для точного позиционирования рамки вокруг переданного children можно использовать хук measuring.md, если размер неизвестен заранее, или требовать от пользователя передачи явных width / height пропсов для производительности.

Категория 5: Переходы и Камера (Transitions & Camera)

Название: <DirectionalWipe /> (и Slide/Pan)
Короткое описание: Классический сдвиг экрана.
Подробное описание эффекта: Смена сцен сдвигом.
Remotion implementation:
Transitioning: Две сцены оборачиваются в пересекающиеся <Sequence>.
Transforms: Согласно документации, используем только transform: translateX/Y, а не left/top, чтобы не триггерить пересчет Layout (Reflow) в браузере при рендере кадров.

Название: <DeviceMockupZoom />
Короткое описание: Отдаление от UI к устройству.
Подробное описание эффекта: Интерфейс отдаляется и оказывается экраном ноутбука/телефона.
Remotion implementation:
Рамки мокапов (png/webp с прозрачным экраном) загружаем через staticFile() (assets.md).
Интерфейс пользователя помещается "под" рамку (z-index или порядок в DOM). Затем весь родительский контейнер с мокапом анимируется через transform: scale().

Название: <ZoomThroughTransition />
Короткое описание: Пролет "сквозь" элемент.
Подробное описание эффекта: Резкое увеличение сцены.
Remotion implementation: Интерполяция scale с сильным ускорением (экспоненциальным или с помощью агрессивного easing). Крайне важно использовать transform-origin, нацеленный точно на центр элемента, "сквозь" который мы пролетаем.

Категория 6: Визуализация данных (Data Visualization)

Название: <AnimatedLineChart /> / <AnimatedBarChart />
Короткое описание: Оживающие графики для дашбордов.
Подробное описание эффекта: Линия графика плавно отрисовывается слева направо. Столбцы гистограммы упруго "вырастают" снизу вверх. Цифры на осях могут "накручиваться" (Count-up).
Remotion implementation: * Линии: Использование SVG <path>. Анимация достигается через интерполяцию свойства stroke-dashoffset от длины пути (можно посчитать заранее или через getTotalLength() в useEffect) до 0.
Столбцы: spring анимация scaleY для <rect>, с обязательным transform-origin: bottom.
Математика: Использование remotion/animation-math для плавного накручивания чисел (count-up).

Категория 7: Специфичные Dev-блоки (Dev & CLI)

Название: <TerminalSimulator />
Короткое описание: Окно консоли с выводом логов.
Подробное описание эффекта: Отличается от <Typewriter>. Здесь симулируется выполнение команд (например, npm run build): текст появляется строками, а когда окно заполняется, старые строки "уезжают" вверх, освобождая место для новых.
Remotion implementation: * Массив объектов { text: string, type: 'command' | 'log' | 'success', delay: number }.
Отрисовка новых строк через <Sequence>.
Симуляция скролла: родительский контейнер сдвигается вверх (transform: translateY) на высоту строки каждый раз, когда добавляется новая строка за пределами видимости.

Название: <CodeDiffWipe /> (Before/After)
Короткое описание: Сравнение "До" и "После".
Подробное описание эффекта: На экране старый, громоздкий код (или UI). Поверх него плавно скользит вертикальная или диагональная линия (часто с ползунком), открывая новый, чистый код (или результат работы продукта).
Remotion implementation: * Два слоя (компонента), наложенных друг на друга (position: absolute).
Верхний слой анимируется через CSS clip-path: inset(0 X% 0 0) (или polygon), где X интерполируется от 100 до 0 с помощью spring или плавной interpolate.

Категория 8: Микроинтеракции (Micro-interactions)

Название: <ToastNotification />
Короткое описание: Всплывающее системное уведомление.
Подробное описание эффекта: Небольшая плашка (например, "Deployment successful") выскакивает в углу экрана, висит несколько секунд и исчезает.
Remotion implementation:
Компонент с абсолютным позиционированием (например, bottom-4 right-4).
Появление через spring (translateY вверх + opacity).
Исчезновение через interpolate (translateY вниз + opacity), используя useCurrentFrame() относительно конца длительности компонента (вычисляется из переданного пропса durationInFrames).

Название: <DragAndDropFlow />
Короткое описание: Симуляция загрузки файла.
Подробное описание эффекта: Файл (иконка документа/картинки) "перетаскивается" курсором в зону загрузки, зона подсвечивается (меняет границу/цвет), файл отпускается, и начинается анимация загрузки (прогресс-бар).
Remotion implementation:
Объединение <SimulatedCursor> и компонента зоны загрузки.
Синхронизация по кадрам: когда координаты курсора совпадают с зоной загрузки, зона использует interpolateColors для подсветки. После "клика" (отпускания) запускается Sequence с анимацией <PulsingIndicator> или растущего width у прогресс-бара.

Категория 9: Процессы и Статусы (Workflows)

Название: <ProgressSteps /> / <Timeline />
Короткое описание: Визуализация многошагового процесса.
Подробное описание эффекта: Горизонтальная или вертикальная линия с точками (шагами). Точки последовательно "зажигаются" (меняют цвет), а линия между ними "заполняется" цветом, показывая движение данных или процесс сборки (например, "Connect -> Process -> Deploy").
Remotion implementation:
Использование <Sequence> для координации старта каждого шага.
Линии между шагами — это SVG <line>, которые закрашиваются через анимацию stroke-dashoffset (как в графиках).
Сами точки пульсируют (spring scale) при переходе в активное состояние.

Название: <SuccessConfetti />
Короткое описание: Радостный финал успешного действия.
Подробное описание эффекта: Взрыв конфетти или партиклов из определенной точки (например, после нажатия кнопки "Merge" или "Publish").
Remotion implementation:
В документации Remotion есть отличные примеры работы с canvas для частиц.
Обязательное условие: использование random(seed) из @remotion/random для генерации векторов полета частиц. Обычный Math.random() сломает консистентность при рендере видео. Партиклы рендерятся в <canvas> внутри компонента Remotion, где координаты пересчитываются на каждый кадр.
